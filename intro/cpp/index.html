    C++ in Competitive Programming

    C++ is the most used language in CP because of its efficiency. So, I will be mostly using C++ in sample codes but the thoughts apply to any other prgramming languages.

    Knowledge on pointer, struct, and class are not required for competitive programming.

    General Template

    This template is what I personally use to shorten my code and also speed up I/O (input and output).

    When the size of problem \(n\) becomes larger than 1M, I/O can be an obstacle in the program's runtime. In these scenarios, speeding up I/O is really necessary to pass in the given time.

    Templates can also save your time in practice so you don't have to type everything down before actually solving the problem

    Tools to use

    In CP, we often use light-weighted IDE or even editor like Vim / Sublime Text. Heavy weighted IDE like Visual Studio or Eclipse can be used but they are designed for larger projects and can be an overkill since the whole program will be in a single source file.

    Often used tools include:

    • Vim / Emacs: classic editor, also check Neovim, Macvim.

    • Sublime Text 3: fast, lightweight, free to use.

    • Geany: lightweight

    • VSCode: PAP and VSCode Docs

    • CP Editor: editor designed for CP.

    STL

    Standard template library is one of the reasons why C++ is widely used in CP. C++ STL offers many helpful functions and data types.

    These functions and data types prevent us from implementing a BST in real contest mannually so it is helpful to take a look at STL while learning some of the basic algorithms.
